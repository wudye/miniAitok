DTO 是 Data Transfer Object（数据传输对象）的缩写。简单说明：
作用：在应用不同层或进程之间传递数据的简单对象，通常用于控制器/服务/持久层之间或客户端与服务端之间的数据交换。
特点：只包含字段和访问器（getter/setter），不包含业务逻辑；通常可序列化并带有用于校验或序列化的注解（例如 validation、JSON 注解）。
优点：隔离领域模型与 API 表现层、只传输需要的字段以减少开销、避免暴露敏感数据、便于做请求/响应校验和映射。

VO（Value Object）解释
VO（Value Object，值对象） 是领域驱动设计（DDD）中的一种核心概念，主要用于封装和传递数据。以下是其核心特点和应用场景：

1. 核心特点
   不可变性（Immutable）：

VO 一旦创建，其内部属性不可修改（通常通过 final 字段和没有 setter 方法实现）。
如果需要修改，会返回一个新的 VO 实例。
基于值的相等性：

两个 VO 的相等性由其所有属性的值决定，而非内存地址（即重写 equals() 和 hashCode() 方法）。
无副作用：

VO 不包含业务逻辑，仅用于存储和传递数据。
2. 与 DTO 的区别
   特性	VO（Value Object）	DTO（Data Transfer Object）
   目的	封装领域内的值	跨层传递数据（如 Controller → Service）
   可变性	不可变	通常可变（可通过 setter 修改）
   业务逻辑	可能包含简单验证逻辑	纯数据容器，无逻辑
   相等性	基于属性值比较	基于内存地址（除非重写方法）
